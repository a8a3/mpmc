
### Async MPMC queue

Неограниченная MPMC очередь.

Содержит 2 внутренние очереди - одна для значений, другая для коллбеков. 
Коллбеки будут исполняться в потоках, вызывающих Enqueue/Dequeue.

В случае закрытия очереди (вызов `Close`) или уничтожения через деструктор, оставшиеся коллбеки вызываются со значением `std::nullopt`.

Послe закрытия очереди вызов `Enqueue` вернет `false`, вызов `Dequeue` будет вызван или с
оставшимся в очереди значений значением или с `std::nullopt` если очередь значений пуста.

Преимущества-
- простота реализации;
- поддерживается FIFO семантика очереди;
- нет накладных расходов на поддержку отельного потока исполнения для коллбеков;

Особенности-
- коллбеки должны быть легковесными (должно быть предусмотрено пользователем);
- время жизни объекта очереди должно превышать время жизни объектов consumer/producer,
  возможно UB в случае одновременной работы деструктора очереди и вложенных внутрь коллбеков
  обращений к интерфейсу очереди;
