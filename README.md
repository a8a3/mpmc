
### Async MPMC queue

Неограниченная MPMC очередь.

Содержит 2 внутренние очереди - одна для значений, другая для коллбеков. 
Коллбеки исполняются в служебном потоке (см. `workerThreadFunc`). 

Для синхронизации используется мьютекс + условная переменная, рабочий поток пробуждается в случае закрытия очереди или если есть возможность вызвать добавленный ранее коллбек (очередь со значениями и очередь с коллбеками не пусты).

В случае закрытия очереди (вызов `Close`) или уничтожения через деструктор, оставшиеся коллбеки вызываются со значением `std::nullopt`.

В случае закрытия очереди вызов `Enqueue` вернет `false`.

Преимущества-
- простота реализации;
- поддерживается FIFO семантика очереди;

Недостатки-
- поставщики и потребитили сериализуются на одном мьютексе;
- все коллбеки последовательно выполняются в одном потоке;
